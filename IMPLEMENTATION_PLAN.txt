PROJECT OVERVIEW
================================================================================
Implementation of two major features in the LMS:
  1. Digital Library System (Books - Borrow & Buy)
  2. Zoom Integration for Live Classes

================================================================================
FEATURE 1: DIGITAL LIBRARY SYSTEM
================================================================================

1.1 DATABASE STRUCTURE (Firestore Collections)
-------------------------------------------

Collection: "books"
  - bookId (String, Primary Key)
  - title (String)
  - author (String)
  - isbn (String)
  - description (Text)
  - category (String)
  - coverImage (URL)
  - totalCopies (Number)
  - availableCopies (Number)
  - borrowPrice (Number) - Monthly rental cost
  - buyPrice (Number)
  - rating (Number) - Average rating 1-5
  - createdDate (Timestamp)
  - updatedDate (Timestamp)

Collection: "borrowTransactions"
  - transactionId (String, Primary Key)
  - studentId (String)
  - bookId (String)
  - borrowDate (Timestamp)
  - dueDate (Timestamp)
  - returnDate (Timestamp, Optional)
  - transactionType (String) - "borrow" or "buy"
  - status (String) - "active", "returned", "overdue", "completed"
  - totalCost (Number)
  - penaltyAmount (Number)

Collection: "bookReviews"
  - reviewId (String, Primary Key)
  - bookId (String)
  - studentId (String)
  - rating (Number) - 1-5
  - reviewText (String)
  - createdDate (Timestamp)
  - helpfulCount (Number)

Collection: "bookNotifications"
  - notificationId (String, Primary Key)
  - studentId (String)
  - bookId (String)
  - type (String) - "return_reminder", "available", "reserved"
  - message (String)
  - dueDate (Timestamp)
  - isRead (Boolean)
  - createdDate (Timestamp)

FEATURE 2: ZOOM CLASS INTEGRATION
================================================================================

2.1 REQUIREMENTS & SETUP
------------------------

Prerequisites:
  - Zoom Account with API credentials
  - Zoom API Key and Secret
  - Zoom Meetings enabled

Integration Points:
  - Schedule classes with Zoom meeting links
  - Generate dynamic meeting URLs for students
  - Store meeting recordings
  - Track attendance

2.2 DATABASE STRUCTURE
---------------------

Collection: "zoomClasses"
  - classId (String, Primary Key)
  - courseId (String) - Link to course
  - professorId (String)
  - title (String)
  - description (String)
  - scheduledDate (Timestamp)
  - scheduledTime (String) - "14:00" format
  - duration (Number) - in minutes
  - zoomMeetingId (String)
  - zoomLink (String)
  - recordingUrl (String, Optional)
  - recordingPassword (String, Optional)
  - status (String) - "scheduled", "in-progress", "completed"
  - maxParticipants (Number)
  - createdDate (Timestamp)

Collection: "classAttendance"
  - attendanceId (String, Primary Key)
  - classId (String)
  - studentId (String)
  - joinTime (Timestamp)
  - leaveTime (Timestamp)
  - duration (Number) - in minutes
  - status (String) - "present", "absent", "late"
  - recordedAt (Timestamp)

================================================================================
SECURITY CONSIDERATIONS
================================================================================

1. Access Control:
   - Only authenticated students can access library
   - Only enrolled students can join classes
   - Professors can only see their own classes
   - Admins can manage all books and classes

2. Data Privacy:
   - Zoom recordings stored securely
   - Student attendance data encrypted
   - Review content moderation
   - Personal data protection (GDPR compliant)

3. API Security:
   - Zoom API calls from secure backend
   - Rate limiting on book searches
   - CORS configuration for Zoom links
   - Token refresh for Zoom sessions
